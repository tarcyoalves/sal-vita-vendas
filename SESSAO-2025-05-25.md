### 5. O que ainda precisa ser feito

#### A. Desabilitar auto-suspend do Neon (banco de pedidos)
As abas Carrinhos/Mensagens/Automações carregam lento porque o Neon entra em suspend após inatividade.

**Passo a passo:**
1. Acesse [console.neon.tech](https://console.neon.tech)
2. Selecione o projeto do `ORDERS_DATABASE_URL`
3. Vá em **Settings → Compute**
4. Desative **"Suspend compute after period of inactivity"**
5. Salve

#### B. Corrigir endpoint da Evolution API
O código chama `${WA_SERVER_URL}/send` com `{ phone, message }`, mas a Evolution API padrão usa:

```
POST /message/sendText/{nomeDoInstance}
{ "number": "5511999999999", "text": "Mensagem" }
```

Para corrigir, fornecer:
- Nome da instância configurada na VPS (ex: `sal-vita`)
- Versão da Evolution API (v1 ou v2)

Arquivos a editar:
- `server/routers/recovery.ts` linha 12 — função `sendViaWhatsApp`
- `api/index.ts` linha 384 — cron `abandoned-cart`

### 6. Fluxo completo de pagamento (como está hoje)

```
Cliente acessa premium.salvitarn.com.br
    ↓
Preenche formulário → trackCart (registra abandono + agenda WA 30min)
    ↓
Seleciona frete → createOrder (cria registro no banco)
    ↓
Clica "Pagar" → createPayment (cria preferência MP, retorna init_point)
    ↓
Redirect para Mercado Pago Checkout
    ↓
Pagamento aprovado → MP envia POST para /api/mp-webhook
    ↓
Webhook atualiza paymentStatus = 'confirmed' + cancela automações WA pendentes
    ↓
Cliente volta para /meu-pedido?pedido=ID&status=pago
```
